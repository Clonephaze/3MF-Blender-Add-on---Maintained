API Discovery¶

Other Blender addons can detect and use the 3MF API at runtime. Three strategies are available, from simplest to most robust.

Direct Import (recommended)¶

If you know the addon is installed, a plain try/except is the simplest and most Pythonic approach:

try:
    from io_mesh_3mf.api import import_3mf, export_3mf
except ImportError:
    import_3mf = export_3mf = None

Discovery Functions (from api)¶

is_available() → bool[source]¶

Check if the 3MF API is loaded and available.

If you can import this function, the underlying code is available. This call ensures the module is registered in bpy.app.driver_namespace so that get_api() and the standalone discovery helper (threemf_discovery) can find it.

Returns False only when the addon has been explicitly disabled via unregister() in the current session.

Returns:

True if the API is ready to use.

Example:

from io_mesh_3mf.api import is_available
if is_available():
    print("3MF API is ready")
get_api()[source]¶

Get the 3MF API module.

Ensures the module is registered in bpy.app.driver_namespace first (handles Blender restart / load-order edge cases), then returns it.

Returns:

The io_mesh_3mf.api module, or None if the addon is disabled.

Return type:

module | None

Example:

from io_mesh_3mf.api import get_api
api = get_api()
if api:
    result = api.import_3mf("/model.3mf")
has_capability(capability: str) → bool[source]¶

Check if a specific API capability is available.

Use this for forward-compatible feature detection instead of version checks. New capabilities may be added in minor versions.

Parameters:

capability – Capability name (e.g., “paint_mode”, “batch”).

Returns:

True if the capability is supported.

Example:

from io_mesh_3mf.api import has_capability
if has_capability("object_settings"):
    # Safe to use object_settings parameter
    result = export_3mf(path, object_settings={...})
check_version(minimum: Tuple[int, int, int]) → bool[source]¶

Check if the API version meets a minimum requirement.

Parameters:

minimum – Tuple of (major, minor, patch) minimum version.

Returns:

True if API_VERSION >= minimum.

Example:

from io_mesh_3mf.api import check_version
if check_version((1, 2, 0)):
    # Use features added in v1.2.0
    ...

Standalone Discovery Helper¶

For addons that want zero runtime dependency on the 3MF addon, copy io_mesh_3mf/threemf_discovery.py into your addon. It resolves the addon’s import path automatically via addon_utils, caches the result, and works regardless of extension repo prefix or addon load order.

3MF API Discovery Helper — Copy this file into your addon.

Provides a single, reliable way to access the 3MF Import/Export addon’s public API from another Blender addon. Works across Blender restarts and regardless of addon load order.

Recommended pattern (put this at the top of your module):

from .threemf_discovery import get_threemf_api

def my_operator_execute(self, context):
    api = get_threemf_api()
    if api is None:
        self.report({'ERROR'}, "3MF Format addon not installed/enabled")
        return {'CANCELLED'}

    result = api.import_3mf("/path/to/model.3mf")
    if result.status == "FINISHED":
        self.report({'INFO'}, f"Imported {result.num_loaded} objects")
    return {'FINISHED'}

Alternative — direct try/except (no helper needed):

try:
    from io_mesh_3mf.api import import_3mf, export_3mf
except ImportError:
    import_3mf = export_3mf = None

Both approaches survive Blender restarts. The helper adds automatic addon-path resolution via addon_utils when a direct import isn’t possible (e.g. the extension repo prefix varies between installs).

get_threemf_api() → None[source]¶

Return the 3MF API module, or None if the addon isn’t available.

This is the recommended single entry point. It resolves the addon’s import path automatically, caches the result, and ensures the module is registered for fast subsequent lookups.

Returns:

The io_mesh_3mf.api module, or None.

Example:

api = get_threemf_api()
if api:
    result = api.import_3mf("/model.3mf")
is_threemf_available() → bool[source]¶

Check whether the 3MF addon is installed and enabled.

Equivalent to get_threemf_api() is not None but reads better in boolean contexts.

Returns:

True if the 3MF API can be used.

get_threemf_version() → Tuple[int, int, int] | None[source]¶

Return the 3MF API version tuple (major, minor, patch), or None.

Returns:

Version tuple like (1, 0, 0), or None if unavailable.

check_threemf_version(minimum: Tuple[int, int, int]) → bool[source]¶

Check if the installed 3MF API meets a minimum version requirement.

Parameters:

minimum – Tuple of (major, minor, patch) minimum version.

Returns:

True if the API version >= minimum, False otherwise.

Example:

if check_threemf_version((1, 2, 0)):
    # Safe to use features added in v1.2.0
    ...
has_threemf_capability(capability: str) → bool[source]¶

Check if a specific API capability is supported.

Use this for forward-compatible feature detection instead of version checks. New capabilities may be added in minor versions.

Capabilities include:

  • "import", "export", "inspect", "batch"

  • "callbacks" (on_progress, on_warning, on_object_created)

  • "target_collection", "orca_format", "prusa_format"

  • "paint_mode", "project_template", "object_settings"

  • "building_blocks", "global_scale", "compression"

  • "thumbnail", "use_components", "auto_smooth"

  • "subdivision_depth"

Parameters:

capability – Capability name string.

Returns:

True if the capability is supported.
